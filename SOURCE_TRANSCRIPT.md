# Greg Isenberg × Dan Koe: Content Operating System

## Podcast Notes — 2026

**Source:** Greg Isenberg podcast
**Guest:** Dan Koe (creator, millions of followers)
**Topic:** How to build a content machine with LLMs + smart systems

---

## Key Insights

### The Core System

1. **Write Twitter first** — 280 chars is the limiting factor. If it works there, it works everywhere.
2. **2-3 social posts per day** minimum
3. **1 newsletter per week** baseline
4. **One piece of content → repurposed across all platforms**

### The Repurposing Chain

```
Newsletter (1x/week)
        ↓
Tweet (2-3x/day) ← START HERE
        ↓
┌──────────────────────────────────────┐
│  Threads → Instagram → LinkedIn      │
│  Reels/TikToks (tweet as script)    │
└──────────────────────────────────────┘
```

### Dan Koe's Tools

- Just Claude and ChatGPT (no fancy stack)
- Simple system, not complex tools
- "The secret is LLMs, AI, and prompts"

### Content Principles

- All content is the same across platforms
- Twitter is the anchor — write it first
- Don't edit while generating — generate first, then edit
- Your voice > perfect voice
- Authenticity compounds

### What Success Looks Like

- Millions of followers
- High output without feeling like it's high output
- System handles the heavy lifting
- AI augments, doesn't replace thinking
